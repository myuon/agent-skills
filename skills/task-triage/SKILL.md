# Task Triage

タスク一覧を「リファイン必要」と「実装着手可能」に振り分け、それぞれ処理する。

## リファインの定義

リファインとは、タスクの要件を実装着手可能な状態まで具体化すること。以下をすべて満たせばリファイン済みとする:

- 何を作る・変更するかが一意に解釈できる
- 完了条件が明確である
- ユーザーの判断待ちの分岐がない

リファイン未完了のタスクは、ユーザーと対話して上記を満たすまで詳細化する。

## 実装着手可能なタスクの処理

リファイン済みのタスクは Agent ツールで並列に実装する。各 Agent には以下を指示:

- `isolation: "worktree"` を指定して worktree 上で作業させる
- 対象 Issue の番号・内容・完了条件
- 実装後に commit & push し、`Closes #<number>` を含む PR を作成すること
